# 页面设计文档（桌面优先）

## 全局设计（适用于全站）

### Layout
- 桌面优先：以 1200px 内容最大宽度为基准，左右留白自适应；核心内容区居中。
- 结构：页面整体为纵向堆叠（Header + Content），Content 内含横向滚动区域。
- 横向滚动：采用 `overflow-x: auto; scroll-snap-type: x mandatory;`，每个板块为一个 snap 点；板块间使用固定 gutter。

### Meta Information
- Title：按现有页面 Title（不在本需求范围内变更）。
- Description：按现有页面 description（不在本需求范围内变更）。
- Open Graph：保持现有配置不变。

### Global Styles（Design Tokens）
> 以下 token 用于同时生成明/暗主题；建议用 CSS variables 或 Tailwind theme 映射。

**颜色（Light）**
- `--bg`: #FFFFFF
- `--surface`: #F7F8FA（卡片底）
- `--text`: #111827
- `--text-muted`: #6B7280
- `--border`: rgba(17,24,39,0.10)
- `--accent`: 按现有品牌色（不改动，仅复用）

**颜色（Dark）**
- `--bg`: #0B0F19
- `--surface`: #111827
- `--text`: #E5E7EB
- `--text-muted`: #9CA3AF
- `--border`: rgba(229,231,235,0.12)
- `--accent`: 品牌色保持一致；必要时降低饱和/提高明度以保证对比度

**排版**
- 字体：沿用现有字体栈
- 标题（Title）：24–28px / 600
- 区块标题：16–18px / 600
- 正文：14–16px / 400

**按钮/可点击态**
- 默认：使用 `--surface` 背景 + `--border`
- Hover：背景提升一档（Light 更浅灰；Dark 更亮一档 surface），并加轻微阴影
- Pressed：轻微缩放（0.98）或降低亮度
- Focus：2px outline，颜色使用 `--accent`，保证键盘可达

**暗色模式策略（自动生成）**
- 默认跟随系统：`prefers-color-scheme`。
- 若产品已有主题开关：开关优先级高于系统；本次仅要求“方案”，实现可后续落地。
- 图片/插画：若存在纯黑图标，需提供反色版本或使用 `currentColor` SVG。

## 页面：主页面

### Page Structure
1. 顶部区（Header，保持不变）
2. 内容区（Content）
   - 横向滚动三板块（Board 1 / Board 2 / Board 3）

### 1) 顶部区（Header，不变）
- 组成：Mark（左）+ Title（右或下，按现有版式）。
- 行为：无新增交互；在暗色模式下仅做颜色适配（文字/图标对比度）。

### 2) 横向滚动三板块（核心改造）

#### 容器
- 宽度：占满内容区宽度。
- 交互：
  - 鼠标滚轮横向滚动（可选：按住 Shift 横向滚动）。
  - 触控板左右滑动。
  - `scroll-snap` 保证停留在整板块。
- 反馈：按 Figma 显示板块指示（分页点/数字/进度条，任选其一以 Figma 为准）。

#### 板块通用（Board 1/2/3）
- 外观：卡片容器（`--surface` 背景、圆角 12、1px `--border`），内边距 16–20。
- 阴影：Light 轻阴影；Dark 阴影更弱、更多依赖边框分层。
- 内容滚动：板块内部允许纵向滚动（若内容溢出），但优先控制高度避免双滚动。

### 2.1 板块 1（包含：推荐 + 成都玩法列表 + 可展开月视图日历）

#### A. 推荐（Recommendation）
- 位置：板块 1 顶部区域。
- 结构：
  - 区块标题（例如“推荐”）
  - 推荐卡片（1–2 张可见；多则横向小滚动或分页点，按 Figma）
- 卡片内容：封面/图标 + 标题 + 简短描述（如 Figma 有）。
- 状态：默认 / hover / pressed / focus；暗色模式下文字与封面边框适配。

#### B. 成都玩法列表（Chengdu Play List）
- 位置：推荐下方。
- 结构：
  - 区块标题（例如“成都玩法”）
  - 列表项：左侧小图标/标签（如有）+ 主标题 + 次要信息（可选）
- 交互：
  - 列表可滚动浏览。
  - 点击列表项：若无独立详情页，则使用“右侧抽屉/居中弹层”展示更多信息（标题、正文、关闭按钮）；此弹层同样适配暗色模式。

#### C. 可展开月视图日历（Expandable Month Calendar）
- 位置：板块 1 底部。
- 折叠态（默认）：
  - 展示当前月份摘要（例如“2026年4月”）+ “展开”按钮。
  - 可选：展示已选日期或今日提示。
- 展开态（月视图）：
  - 顶部：月份标题 + 上/下月切换（如 Figma 有）+ “收起”。
  - 主体：7 列网格（周一/周日是否为首列按 Figma）；日期格子为按钮。
  - 选中态：使用 `--accent` 描边或底色；文字保持可读对比。
  - 今日态：弱强调（描边/小点）。
- 动效：展开/收起使用 150–220ms 高度过渡 + 淡入；避免影响横向滚动手感。

## 响应式与可用性
- 桌面优先；当宽度 < 768px：板块宽度改为接近视口宽（例如 90vw），仍保持横向滑动与 snap。
- 键盘可达：
  - 横向区域允许 Tab 进入板块内元素。
  - 日历日期格子必须可聚焦与回车选择。
- 对比度：暗色模式下正文与背景对比度优先保证可读性。
